## Abstract
AI has fundamentally changed how engineers learn. Previous generations relied on books, colleagues, Stack Overflow, documentation, and online courses. Today's engineers have an assistant that can generate explanations, designs, and working code in an instant.

How do you know whether someone understands the code they generated? What kind of feedback do you give on work that is mostly AI generated? And how do you encourage AI usage without letting it replace learning?

I want to stop focussing on reviewing only code and start reviewing the process that produced it. I will share practical tips for reviewing chat histories and how to encourage Engineers to use AI in ways that make it easier to learn instead of using it as an answer machine.

You'll leave with practical ideas you can immediately apply when mentoring engineers.
